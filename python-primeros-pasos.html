1. Sintaxis básica

Python es sensible a la indentación (espacios o tabuladores). Un bloque se define por la indentación.

# Bloque con indentación consistente
if True:
    print("Hola")
    x = 5
    if x > 2:
        print("x es mayor que 2")

Consejo: usa 4 espacios por nivel (PEP8 estándar). No mezcles tabs y espacios.

2. Cómo nombrar variables y funciones

  • Usar nombres descriptivos: contador, nombre_alumno
  • No empezar con número: 3d es inválido
  • No usar caracteres especiales (excepto _)
  • Convención: snake_case para variables y funciones (mi_funcion); CapWords para clases (MiClase).
  • No usar palabras reservadas (ver sección 8).
# Correcto
nombre_alumno = "Emilio"
# Incorrecto
# 2nombre = 10

6. Funciones y parámetros

Definición con def. Pueden tener parámetros posicionales, por defecto, con nombre y argumentos variables.

def saludar(nombre):
    print('Hola', nombre)

# Parámetro por defecto
def potencia(x, n=2):
    return x ** n

# Args / Kwargs
def ejemplo(*args, **kwargs):
    print(args, kwargs)

def sumar(a, b=0):
    return a + b

print(sumar(3))    # 3
print(sumar(3,4))  # 7

7. Imprimir información

Usar print(). F-strings (Python 3.6+) son recomendadas para legibilidad.

nombre = 'Ana'
edad = 30
print('Nombre:', nombre, 'Edad:', edad)
print(f"{nombre} tiene {edad} años")

9. Errores comunes y cómo arreglarlos

IndentationError 
  File "prog.py", line 3
    print('hola')
IndentationError: unexpected indent

Solución: mantener indentación consistente (4 espacios).

SyntaxError 
  File "prog.py", line 1
    print('hola'
                ^
SyntaxError: unexpected EOF while parsing

Solución: faltó un paréntesis, comillas o dos puntos. Revisa la sintaxis alrededor de la línea indicada.

NameError 
NameError: name 'x' is not defined

Solución: la variable no existe en ese scope. Define antes o revisa el nombre.

TypeError 
TypeError: can only concatenate str (not "int") to str

Solución: convertir tipos (ej. str() o usar f-strings) o validar tipos.

IndexError / KeyError 
IndexError: list index out of range
KeyError: 'id'

Solución: comprobar índices y claves antes de acceder (condicionales, in, dict.get()).

Errores de novato y buenas prácticas

  • No usar == cuando quieres asignar (en Python la asignación es =, no hay confusion con ==). En otros lenguajes sí hay confusión; revisen siempre el operador.
  • No mezclar tabs y espacios.
  • No modificar listas mientras se itera sobre ellas (puede dar resultados inesperados).
  • Evitar usar variables globales salvo necesario.

10. Ejercicios (con respuestas ocultas)

  1. Escribir una función par_o_impar(n) que devuelva "par" o "impar".
  2. Dada una lista de números, devolver la suma de los pares.
  3. Crear una función que reciba **kwargs y muestre las claves y valores en líneas nuevas.
Soluciones (haz clic para ver) 
# 1
def par_o_impar(n):
    return 'par' if n % 2 == 0 else 'impar'

# 2
def suma_pares(lista):
    return sum(x for x in lista if x % 2 == 0)

# 3
def muestra_kwargs(**kwargs):
    for k, v in kwargs.items():
        print(f"{k}: {v}")
