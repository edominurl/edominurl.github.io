Parte 1: Persistencia de Datos con JSON (3 horas)

Hasta ahora, los datos en nuestros programas desaparecen cuando el programa termina. La persistencia de datos nos permite:

  • Guardar información entre ejecuciones del programa
  • Compartir datos entre diferentes partes de la aplicación
  • Mantener registros históricos
  • Trabajar con grandes volúmenes de información

JSON (JavaScript Object Notation) es un formato ligero para intercambio de datos, muy usado en aplicaciones web.

Python JSON Tutorial (en español) - Programación en Python
1. Estructura de archivos JSON: 
{
    "nombre": "María López",
    "edad": 16,
    "notas": [85, 90, 78],
    "activo": true
}
2. Módulo json en Python: 
import json

# Guardar datos en JSON (dump)
datos = {"nombre": "Juan", "edad": 15}
with open("datos.json", "w", encoding="utf-8") as archivo:
    json.dump(datos, archivo, indent=4, ensure_ascii=False)

# Leer datos desde JSON (load)
with open("datos.json", "r", encoding="utf-8") as archivo:
    datos_leidos = json.load(archivo)
3. Trabajar con listas de diccionarios: 
estudiantes = [
    {"id": 1, "nombre": "Ana", "promedio": 85},
    {"id": 2, "nombre": "Carlos", "promedio": 78},
    {"id": 3, "nombre": "María", "promedio": 92}
]

# Guardar lista completa
with open("estudiantes.json", "w") as f:
    json.dump(estudiantes, f, indent=4)

# Buscar un estudiante específico
for est in estudiantes:
    if est["nombre"] == "María":
        print(f"Promedio de María: {est['promedio']}")

Parte 2: Bases de Datos con SQLite (3.5 horas)

Aunque JSON es útil, las bases de datos ofrecen ventajas importantes:

  • Eficiencia: Consultas rápidas en grandes volúmenes de datos
  • Integridad: Validación automática de datos
  • Relaciones: Conexión entre diferentes tipos de información
  • Concurrencia: Múltiples usuarios accediendo simultáneamente

SQLite es una base de datos completa contenida en un solo archivo, perfecta para aprender y para aplicaciones pequeñas.

Python SQLite Tutorial (en español) - Base de datos con Python
1. Conexión a SQLite: 
import sqlite3

# Conectar (crea el archivo si no existe)
conexion = sqlite3.connect('mi_base_datos.db')
cursor = conexion.cursor()

# Siempre cerrar al final
conexion.close()
2. Crear tabla (CREATE TABLE): 
cursor.execute('''
    CREATE TABLE IF NOT EXISTS estudiantes (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        nombre TEXT NOT NULL,
        grado TEXT,
        promedio REAL,
        ausentismo INTEGER,
        fecha_registro DATE
    )
''')
conexion.commit()
3. Insertar datos (INSERT): 
# Insertar un estudiante
cursor.execute('''
    INSERT INTO estudiantes (nombre, grado, promedio, ausentismo, fecha_registro)
    VALUES (?, ?, ?, ?, ?)
''', ("María López", "5to Primaria", 85.5, 2, "2025-01-15"))
conexion.commit()

# Insertar múltiples estudiantes
estudiantes = [
    ("Carlos Pérez", "6to Primaria", 78.0, 5, "2025-01-15"),
    ("Ana Martínez", "5to Primaria", 92.0, 1, "2025-01-15")
]
cursor.executemany('''
    INSERT INTO estudiantes (nombre, grado, promedio, ausentismo, fecha_registro)
    VALUES (?, ?, ?, ?, ?)
''', estudiantes)
conexion.commit()
4. Consultar datos (SELECT): 
# Obtener todos los estudiantes
cursor.execute('SELECT * FROM estudiantes')
todos = cursor.fetchall()
for estudiante in todos:
    print(estudiante)

# Buscar estudiantes específicos
cursor.execute('SELECT * FROM estudiantes WHERE promedio < 60')
en_riesgo = cursor.fetchall()

# Buscar por nombre
cursor.execute('SELECT * FROM estudiantes WHERE nombre LIKE ?', ('%María%',))
resultado = cursor.fetchone()

Parte 3: Templates Dinámicos con Jinja2 (2 horas)

Hasta ahora hemos retornado HTML directamente desde Flask. Los templates nos permiten:

  • Separar la lógica de Python de la presentación HTML
  • Reutilizar estructura HTML en múltiples páginas
  • Mostrar datos dinámicos (listas, condicionales, loops)
  • Mantener el código organizado y fácil de mantener

Jinja2 es el motor de templates que usa Flask.

Flask Templates y Jinja2 (en español) - Código Facilito
1. Estructura de carpetas: 
mi_proyecto/
├── app.py
├── templates/
│   ├── inicio.html
│   └── estudiantes.html
└── estudiantes.db
2. Renderizar templates desde Flask: 
from flask import Flask, render_template
import sqlite3

app = Flask(__name__)

@app.route('/')
def inicio():
    # Obtener datos de la base de datos
    conexion = sqlite3.connect('estudiantes.db')
    cursor = conexion.cursor()
    cursor.execute('SELECT * FROM estudiantes')
    estudiantes = cursor.fetchall()
    conexion.close()
    
    # Pasar datos al template
    return render_template('inicio.html', estudiantes=estudiantes)

if __name__ == '__main__':
    app.run(debug=True)
3. Usar variables en templates (inicio.html): 
<!DOCTYPE html>
<html>
<head>
    <title>Sistema de Estudiantes</title>
</head>
<body>
    <h1>Lista de Estudiantes</h1>
    <p>Total de estudiantes: {{ estudiantes|length }}</p>
    
    <table>
        <tr>
            <th>ID</th>
            <th>Nombre</th>
            <th>Promedio</th>
            <th>Estado</th>
        </tr>
        {% for estudiante in estudiantes %}
        <tr>
            <td>{{ estudiante[0] }}</td>
            <td>{{ estudiante[1] }}</td>
            <td>{{ estudiante[3] }}</td>
            <td>
                {% if estudiante[3] < 60 %}
                    <span style="color: red;">En Riesgo</span>
                {% elif estudiante[3] < 75 %}
                    <span style="color: orange;">Alerta</span>
                {% else %}
                    <span style="color: green;">Normal</span>
                {% endif %}
            </td>
        </tr>
        {% endfor %}
    </table>
</body>
</html>
4. Sintaxis de Jinja2 más común:
  • {{ variable }} - Mostrar valor de variable
  • {% for item in lista %} ... {% endfor %} - Bucle
  • {% if condicion %} ... {% endif %} - Condicional
  • {{ lista|length }} - Filtros (contar elementos)
  • {{ texto|upper }} - Convertir a mayúsculas

Modifica tu proyecto Flask para:

  1. Crear una carpeta templates/
  2. Mover tu HTML a un archivo templates/inicio.html
  3. Usar render_template() en lugar de retornar HTML directo
  4. Pasar la lista de estudiantes al template
  5. Usar un loop {% for %} para mostrar todos los estudiantes

Parte 4: Reflexión Pedagógica - Ética en Datos (1 hora)

¿Qué consideraciones éticas deben tenerse al manejar datos personales de estudiantes?

Como futuros docentes que manejarán información sensible de estudiantes, es fundamental reflexionar sobre:

  • Privacidad y confidencialidad de datos personales
  • Consentimiento para recolección y uso de información
  • Seguridad en el almacenamiento y transmisión de datos
  • Uso responsable de información para fines educativos
  • Consecuencias de filtraciones o mal uso de datos

Escribe un texto reflexivo de 300-400 palabras que incluya:

  1. Análisis ético: ¿Qué datos personales estamos manejando en nuestro sistema? ¿Qué riesgos existen?
  2. Código de ética: Propón 5 principios básicos que seguirías al manejar este sistema en una escuela real
  3. Aplicación pedagógica: ¿Cómo enseñarías a tus futuros estudiantes (niños/adolescentes) sobre privacidad y protección de datos de manera práctica?
  4. Reflexión personal: ¿Qué aprendiste al pensar en estas consideraciones éticas?
